# 部署通知脚本重试机制修复报告

**修复日期**: 2026-02-02
**修复文件**: `deploy/send_deploy_notification.py`

---

## 📋 问题描述

### 原始问题

部署通知邮件中的播客源检查出现大量失败，但实际运行时播客源可以正常抓取。

### 根本原因

1. **超时时间过短**: 原超时设置为8秒，对于跨国网络请求太短
2. **无重试机制**: 临时网络波动会导致检查失败
3. **误报误导用户**: 部署通知显示失败，但实际运行正常

---

## 🔧 修复方案

### 修改内容

#### 1. 添加 `time` 模块导入

```python
import time  # 新增
```

#### 2. 增强 `check_url` 函数签名

**原函数**:
```python
def check_url(url: str, timeout: int = 10) -> Tuple[bool, str]
```

**新函数**:
```python
def check_url(
    url: str,
    timeout: int = 10,
    max_retries: int = 2,
    retry_delay: int = 5
) -> Tuple[bool, str]
```

**新增参数**:
- `max_retries`: 最大重试次数（默认2次）
- `retry_delay`: 重试延迟时间（默认5秒）

#### 3. 实现重试逻辑

```python
def check_url(url: str, timeout: int = 10, max_retries: int = 2, retry_delay: int = 5) -> Tuple[bool, str]:
    """检查 URL 是否可访问（支持重试）"""

    for attempt in range(max_retries + 1):
        try:
            # ... 尝试连接 ...
            if response.status == 200:
                if attempt > 0:
                    return True, f"正常（重试{attempt}次后成功）"
                return True, "正常"
            return False, f"HTTP {response.status}"

        except Exception as e:
            if attempt < max_retries:
                time.sleep(retry_delay)  # 延迟后重试
                continue
            return False, str(e)[:20]

    return False, "已达最大重试次数"
```

**重试流程**:
1. 第1次尝试：立即执行
2. 如果失败：等待5秒
3. 第2次尝试：执行
4. 如果仍失败：等待5秒
5. 第3次尝试（最后）：执行
6. 返回最终结果

**总计**: 最多3次查询，每次失败后延迟5秒重试

#### 4. 更新调用参数

**播客源检查**:
```python
# 原来
ok, msg = check_url(url, timeout=8)

# 现在
ok, msg = check_url(url, timeout=15, max_retries=2, retry_delay=5)
```

**投资数据源检查**:
```python
# 原来
ok, msg = check_url(src["url"], timeout=8)

# 现在
ok, msg = check_url(src["url"], timeout=15, max_retries=2, retry_delay=5)
```

## 💡 设计考量

### 为什么选择5秒延迟？

1. **避免过快重试**: 立即重试通常也会失败（网络问题需要恢复时间）
2. **合理等待**: 5秒足够临时网络问题恢复
3. **总时长可控**: 15秒×3 + 5秒×2 = 55秒，可接受

### 为什么重试2次（总共3次）？

1. **平衡准确性和速度**: 太多次重试浪费时间，太少则无法处理临时问题
2. **实践经验**: 大部分临时网络问题在1-2次重试内恢复
3. **与实际运行一致**: 实际播客抓取器也有超时和错误处理

### 为什么超时15秒？

1. **跨国网络**: 大部分播客源托管在国外，需要更长时间
2. **包含SSL握手**: HTTPS连接需要额外时间
3. **DNS解析**: 首次连接需要DNS查询时间
