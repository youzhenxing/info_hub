# 163邮箱SMTP认证失败诊断报告

## 错误详情

```
错误码: 535
错误消息: Error: authentication failed
```

**SMTP调试日志显示**:
```
AUTH PLAIN ... → 535 Error: authentication failed
AUTH LOGIN ... → 535 Error: authentication failed
```

这表明服务器**拒绝了用户名和密码的组合**。

---

## 🔍 可能的原因（按概率排序）

### 1. SMTP服务未开启 ⚠️ **最可能**

163邮箱默认**不开启SMTP服务**，需要手动开启。

**检查方法**:
1. 登录 https://mail.163.com
2. 点击"设置" → "POP3/SMTP/IMAP"
3. 查看"SMTP服务"状态
4. 如果是"未开启"，点击开启

**开启后需要**:
- 可能需要短信验证
- 授权码会自动生成或提示你生成
- 开启后需要等待**5-10分钟**生效

### 2. 授权码类型错误

163邮箱有多种密码类型：

| 密码类型 | 用途 | 用于SMTP |
|---------|------|---------|
| 登录密码 | 登录网页邮箱 | ❌ |
| 客户端授权密码 | 第三方邮件客户端 | ✅ |
| POP3授权密码 | POP3协议 | ❌ |
| IMAP授权密码 | IMAP协议 | ❌ |

**必须使用**: "客户端授权密码"（专门用于SMTP）

### 3. 授权码未生效

新生成的授权码可能需要等待**5-30分钟**才能在SMTP服务器上生效。

### 4. 使用了错误的账号格式

163邮箱SMTP要求使用**完整的邮箱地址**作为用户名：
```
✅ {{EMAIL_ADDRESS}}
❌ youzx163
```

## ✅ 逐步解决流程

### 步骤1: 检查SMTP服务状态

```
1. 登录 https://mail.163.com
2. 设置 → POP3/SMTP/IMAP
3. 找到"SMTP服务"或"发送邮件服务"
4. 查看状态
```

**如果未开启**: 点击"开启"，按提示操作

**如果已开启**: 继续下一步

### 步骤2: 重新生成授权码

即使已经有授权码，也建议重新生成：

```
1. 在同一页面（POP3/SMTP/IMAP设置）
2. 找到"客户端授权密码"
3. 点击"新增授权码"或"重置"
4. 可能需要短信验证
5. **复制新生成的16位授权码**
```

### 步骤3: 等待生效

授权码生成后，等待**10-15分钟**再测试。

### 步骤4: 更新配置

```bash
# 更新环境变量
export EMAIL_PASSWORD="新的16位授权码"

# 或更新配置文件 wechat/config.yaml
email:
  password: "新的16位授权码"
```

### 步骤5: 测试连接

```bash
python3 << 'EOF'
import smtplib

email_from = "{{EMAIL_ADDRESS}}"
email_password = "新的授权码"

try:
    with smtplib.SMTP_SSL("smtp.163.com", 465, timeout=10) as server:
        server.login(email_from, email_password)
        print("✅ 认证成功！")
except Exception as e:
    print(f"❌ 失败: {e}")
EOF
```

## 📞 如果还是不行

### 尝试备用端口

163邮箱可能还支持其他端口：

```python
# 端口25 (不推荐，可能被ISP封禁)
smtplib.SMTP("smtp.163.com", 25)

# 端口994 (SSL备用)
smtplib.SMTP_SSL("smtp.163.com", 994)
```

### 联系163客服

如果以上都确认无误但仍失败，可能是：
- 账号被临时限制
- IP地址被限制
- 服务器维护

## ⚠️ 特别提示

163邮箱对SMTP有限制：
1. **每日发送数量限制** (通常为200封/天)
2. **同一收件人频率限制** (避免被识别为垃圾邮件)
3. **新账号限制** (注册时间较短的限制更多)

如果只是测试，建议：
- 减少发送频率
- 避免短时间内重复发送
- 先发送到自己的另一个邮箱测试
