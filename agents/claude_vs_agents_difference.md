## 🎯 核心区别总结

| 维度 | CLAUDE.md | AGENTS.md |
|------|-----------|-----------|
| **文件性质** | 配置规范文档 | 项目知识库文档 |
| **加载方式** | 自动加载到系统提示 | 需要显式读取 |
| **作用范围** | 通用规则+项目规范 | 详细经验记录 |
| **优先级** | 高（影响所有对话） | 中（作为参考） |

## 📖 详细对比分析

### 1. 文件性质与定位

#### CLAUDE.md - 配置规范文档

**性质**：
- **配置文件**: 定义AI的行为规范和工作方式
- **指令集**: 告诉AI"怎么做"、"遵循什么规则"
- **契约**: 与AI的协作约定

**内容特点**：
- ✅ 规则性: "必须做"、"禁止做"
- ✅ 指导性: 提供具体操作步骤
- ✅ 约束性: 限制AI的行为边界

**示例内容**：
```markdown
# 对话规范
必须使用中文进行对话

# 文件放置规则
所有模型输出的非代码文件必须放置在 agents/ 目录下

# 部署规范
生产环境部署必须使用标准部署流程
```

#### AGENTS.md - 项目知识库文档

**性质**：
- **知识库**: 记录项目的经验教训和最佳实践
- **参考书**: 提供详细的问题解决方案
- **历史记录**: 保存踩坑经验和故障排查

**内容特点**：
- 📚 描述性: "发生了什么"、"为什么发生"
- 🔍 解释性: 深入分析根本原因
- 💡 经验性: 分享实际遇到的问题和解决方案

**示例内容**：
```markdown
## 踩坑经验

### 规则1: 新增独立模块脚本 → 必须同步三处

**历史教训**: run_community.py 新增时仅写了源文件和
entrypoint条目，deploy.sh + pre-deploy-check 全部遗漏 →
社区模块 03:00 cron 必然 crash。

**解决方案**: ...
**验证方法**: ...
**预防措施**: ...
```

## 🤖 Claude Code 的加载机制

### 自动加载（系统提示）

**CLAUDE.md 的加载路径**：

```
会话开始时
  ↓
Claude Code 读取配置
  ↓
1. 全局配置优先
   ~/.claude/CLAUDE.md → 加载到系统提示
  ↓
2. 项目配置其次
   /work/.claude/CLAUDE.md → 如果存在，追加到系统提示
   /work/CLAUDE.md → 如果存在，追加到系统提示
  ↓
3. 其他项目文件
   AGENTS.md → ❌ 不自动加载，需要显式读取
```

**验证证据**：

从本次会话可以看到，系统提示包含：

```markdown
## System: Following are official rules from the user that

<user-intent>
1. 文件放置规则
...
4. 输出html文件时，给出绝对路径
</user-intent>
```

这些规则来自 `~/.claude/CLAUDE.md`，在会话开始时就已经加载。


## 🔍 关键差异详解

### 差异1: 加载时机

| 文件 | 加载时机 | 触发方式 |
|------|----------|----------|
| **CLAUDE.md** | 会话开始时 | 自动加载 |
| **AGENTS.md** | 需要时 | 手动读取（Read工具） |

**实际影响**：
- ✅ CLAUDE.md: 规则始终生效，AI始终遵守
- ⚠️ AGENTS.md: 只有在AI主动读取时才生效

**示例**：
```markdown
# ✅ CLAUDE.md 的规则自动生效
User: "生成一个报告"
AI: 根据 CLAUDE.md，报告放在 agents/ 目录

# ⚠️ AGENTS.md 需要显式读取
User: "为什么我的部署失败了？"
AI: [读取 AGENTS.md] 查看规则0...
    "验证流程没有执行..."
```

### 差异2: 内容优先级

**CLAUDE.md - 高优先级规则**：
```markdown
# 强制要求
⚠️ 生产环境部署必须使用标准部署流程
❌ 禁止手动部署

# 检查清单
- [ ] 使用 deploy/deploy.sh 脚本部署
- [ ] 部署通知邮件已收到
```

**AGENTS.md - 详细经验参考**：
```markdown
# 详细解释
为什么这样做？
- 确保...
- 避免...

# 历史教训
v5.26.0 时发生过...
```

**优先级体现**：
```
用户: "我可以手动部署吗？"

AI 先看 CLAUDE.md:
  ❌ "禁止手动部署" → 直接拒绝

AI 可能再看 AGENTS.md:
  📚 "手动部署仅限开发测试" → 提供上下文

最终决策遵循 CLAUDE.md 的强制规则
```

### 差异3: 更新策略

**CLAUDE.md - 稳定规范**：
```markdown
# 更新频率
- 较低（只在必要时更新）
- 更新需要谨慎（影响所有对话）
- 更新后需要验证（确保规则清晰）

# 更新触发
- 新增通用规范
- 修正规则错误
- 补充遗漏的重要规则
```

**AGENTS.md - 持续更新**：
```markdown
# 更新频率
- 较高（每次遇到问题都要更新）
- 更新可以更自由（记录新经验）
- 更新后立即可用（作为参考）

# 更新触发
- 遇到新问题
- 发现新规律
- 解决新故障
```

**实际示例**：

```markdown
# CLAUDE.md 更新
2026-02-09: 新增"生产环境部署规范"章节
          （这是一个重要的新规范，需要AI始终遵守）

# AGENTS.md 更新
2026-02-09: 新增"Bootstrap 机制版本感知引导"
          （这是一个新的踩坑经验，供参考）
```

## 🎓 最佳实践建议

### 写入 CLAUDE.md 的内容

**应该写入**：
- ✅ 行为规范（对话语言、回复格式）
- ✅ 工作流程（标准部署流程）
- ✅ 强制约束（禁止手动部署）
- ✅ 检查清单（部署前确认项）
- ✅ 快速参考（常用命令、路径）

**不应该写入**：
- ❌ 详细的问题分析（太长）
- ❌ 历史故障记录（参考性质）
- ❌ 临时性工作笔记（不具通用性）
- ❌ 项目特定细节（应放在 AGENTS.md）

### 写入 AGENTS.md 的内容

**应该写入**：
- ✅ 详细的问题分析（问题描述、根本原因）
- ✅ 解决方案步骤（具体命令、代码）
- ✅ 验证方法（如何确认解决）
- ✅ 预防措施（如何避免重复）
- ✅ 历史故障记录（踩坑经验）

**不应该写入**：
- ❌ 基础行为规范（应在 CLAUDE.md）
- ❌ 通用工作流程（应在 CLAUDE.md）
- ❌ 过于简短的条目（缺乏上下文）

### 内容分布示例

**错误示例**：
```markdown
# CLAUDE.md 中写入了太多细节
踩坑经验：
- 规则0: 验证流程（详细说明...）
- 规则1: 脚本同步（详细说明...）
...（11条规则，每条都详细解释）
→ 问题：文件太长，AI不容易记住关键规则

# AGENTS.md 中写入了通用规范
必须使用中文对话 → 应该在 CLAUDE.md
```

**正确示例**：
```markdown
# CLAUDE.md - 简洁有力
## 踩坑经验（必读）
⚡ 规则0: 强制验证流程
⚡ 规则1: 新增脚本→同步三处
...（简要说明，详细内容见 AGENTS.md）

# AGENTS.md - 详细完整
## 规则0: 强制验证流程
**问题描述**: ...
**根本原因**: ...
**解决方案**: ...
**验证方法**: ...
**预防措施**: ...
```
