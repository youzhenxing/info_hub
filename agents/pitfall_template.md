# 踩坑经验记录模板

> 使用此模板记录遇到的问题和解决方案，然后将其添加到 `AGENTS.md` 的"踩坑经验"章节。

---

## 📝 记录模板

```markdown
#### [日期] [问题标题] - 严重程度 🔴/🟡/🟢 - 影响范围

**问题描述**：
- 问题表现（具体行为）
- 影响范围（哪些模块/用户）
- 发现方式（日志、用户反馈、监控）

**根本原因**：
- 技术原因（代码逻辑、配置、依赖）
- 流程原因（文档缺失、检查遗漏）
- 管理原因（沟通不畅、职责不清）

**解决方案**：
```代码/配置/命令```

**验证方法**：
```bash
# 验证命令
```

**预防措施**：
- ✅ 技术措施（代码检查、自动化测试）
- ✅ 流程措施（文档更新、检查清单）
- ✅ 监控措施（告警规则、日志监控）

**相关文档**：
- [文档链接](./path/to/doc)
```

---

## 🎨 严重程度说明

- 🔴 **高**：影响核心功能，导致服务不可用
- 🟡 **中**：影响部分功能，有临时解决方案
- 🟢 **低**：小问题，不影响主要功能

---

## 📋 使用步骤

1. **复制模板**：复制上面的模板到新的 Markdown 文件
2. **填写信息**：按照模板填写完整信息
3. **验证方案**：确保验证方法可以复现问题
4. **添加到 AGENTS.md**：将内容添加到"踩坑经验"章节的相应小节
5. **更新统计**：更新"经验统计"部分的计数和日期
6. **提交代码**：git commit 记录本次更新

---

## 🔍 快速检查清单

在添加新的踩坑经验前，请确认：

- [ ] 问题描述清晰具体（不是模糊的描述）
- [ ] 根本原因已分析（不只是表面现象）
- [ ] 解决方案已验证（确实能解决问题）
- [ ] 验证方法可执行（命令可以直接复制运行）
- [ ] 预防措施明确（有具体的行动项）
- [ ] 相关文档已更新（代码注释、README 等）
- [ ] 影响范围已标注（哪些模块/用户受影响）
