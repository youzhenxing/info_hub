# TrendRadar Docker 部署说明

## 问题说明

如果你遇到从 Docker Hub 拉取镜像失败的问题（即使配置了镜像源），可以使用本地构建的方式来部署。

## 解决方案：本地构建部署

### 方法一：使用构建脚本（推荐）

1. **运行构建脚本**
```bash
cd docker
./build-local.sh
```

2. **启动服务**
```bash
docker-compose -f docker-compose-build.yml up -d
```

3. **查看日志**
```bash
docker-compose -f docker-compose-build.yml logs -f
```

### 方法二：手动构建

1. **构建镜像**
```bash
# 回到项目根目录
cd /home/zxy/Documents/code/TrendRadar

# 构建镜像
docker build -f docker/Dockerfile -t trendradar:local .
```

2. **启动容器**
```bash
cd docker
docker-compose -f docker-compose-build.yml up -d
```

### 方法三：Docker 命令直接运行

```bash
docker run -d \
  --name trendradar \
  --restart unless-stopped \
  -v /home/zxy/Documents/code/TrendRadar/config:/app/config:ro \
  -v /home/zxy/Documents/code/TrendRadar/output:/app/output \
  -e TZ=Asia/Shanghai \
  -e RUN_MODE=cron \
  -e CRON_SCHEDULE="*/30 * * * *" \
  -e IMMEDIATE_RUN=true \
  trendradar:local
```

## 优化说明

已对 Dockerfile 进行了以下优化：

1. **APT 镜像源**：使用清华大学 TUNA 镜像源加速软件包下载
2. **PyPI 镜像源**：使用清华大学 PyPI 镜像源加速 Python 包安装
3. **GitHub 代理**：使用 ghproxy.com 加速 GitHub 文件下载
4. **重试机制**：改进下载重试逻辑，优先使用国内代理

## 常见问题

### Q1: 构建时提示 "no space left on device"
清理 Docker 缓存：
```bash
docker system prune -a
```

### Q2: 构建很慢
这是正常的，首次构建需要下载所有依赖。后续构建会利用缓存加速。

### Q3: 如何查看容器日志
```bash
docker logs trendradar -f
```

### Q4: 如何重启容器
```bash
docker restart trendradar
```

### Q5: 如何停止并删除容器
```bash
cd docker
docker-compose -f docker-compose-build.yml down
```

## 配置文件

确保 `config/config.yaml` 已经配置好。如果没有，请参考项目文档进行配置。

## 下一步

部署成功后，可以：
1. 查看输出目录 `output/` 中的报告
2. 根据需要调整 `config/config.yaml` 配置
3. 查看容器日志确认运行状态
