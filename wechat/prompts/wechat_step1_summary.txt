# 微信公众号文章深度摘要生成提示词

请对以下微信公众号文章生成一个结构化的深度摘要。

## 输出要求

1. **核心观点**（1-2句话）
   - 文章的主要论点或核心结论
   - 作者想要传达的关键信息

2. **关键时间线**
   - 提取文章中提到的所有重要时间节点
   - 包括：事件发生时间、产品发布日期、政策生效日期、财报周期等
   - 格式：「YYYY年MM月」或「YYYY年Q1」等

3. **关键信息**（3-5点）
   - 重要的数据、统计、百分比
   - 具体的事件、公司、产品
   - 金额、人物、地点等关键要素

4. **案例详解**
   - 如果文章包含具体案例、实例或故事，需要详细展开
   - 说明案例背景、关键人物/公司、具体做法、结果数据
   - 每个案例 50-100 字

5. **延伸思考**（1-2句话）
   - 为什么这篇文章值得关注
   - 对读者有什么启发或价值

## 格式规范

- 摘要总长度控制在 **300-500 字**
- 保留原文中的 **具体数据** 和 **关键术语**
- 使用 **中文** 输出
- 不要添加 Markdown 格式标记

## 示例输出

核心观点：OpenAI 发布 GPT-5，性能较 GPT-4 提升 40%，重新定义了大模型能力边界，可能改变企业软件市场格局。

关键时间线：
• 2025年3月15日：GPT-5 正式发布
• 2025年Q2：企业版 API 开放
• 2025年下半年：预计推动新一轮 AI 应用潮

关键信息：
• 训练成本：约20亿美元，较 GPT-4 增加 3 倍
• 支持100万 token 上下文窗口（GPT-4 为 128K）
• 首次实现多模态原生融合
• API 定价降低 50%，月活用户突破 5 亿

案例详解：
文章提到微软 Copilot 接入 GPT-5 后的效果。微软在内部测试中让 500 名工程师使用新版 Copilot 进行代码开发，结果显示代码编写效率提升 65%，Bug 率下降 40%。特别是在复杂系统重构任务中，原本需要 2 周的工作缩短至 3 天完成。

延伸思考：这标志着 AGI 进程的重大突破，各行业需要重新评估 AI 应用策略，特别是软件开发、客服、内容创作等领域将迎来颠覆。
