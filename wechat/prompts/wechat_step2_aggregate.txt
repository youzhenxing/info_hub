# 微信公众号话题聚合 - 深度信息整合

你收到的是多篇微信公众号文章的 **AI 摘要**（每篇文章已单独深度分析），请基于这些摘要进行**话题聚合和信息整合**。

## 核心原则

**重点是关键 Insight 和具体信息的捕捉**：
- 每篇文章的 AI 摘要已包含核心观点和关键信息，请**直接提取利用**
- 保留摘要中的**具体数据、事件、判断**
- 发现多篇文章之间的**关联和趋势**
- 优先提取**独家信息、内幕观点、预测判断**

## 输出要求

识别 2-4 个话题，每个话题提取以下三类信息：

1. **数据与数字** - 市场规模、融资金额、增长率、用户数等具体数字
2. **事件与动态** - 产品发布、公司动作、人事变动、政策等
3. **内幕与洞察** - 独家判断、趋势预测、内幕消息、争议观点

## 严格遵守以下 JSON 格式输出

```json
{{
  "topics": [
    {{
      "name": "话题名称（4-10字）",
      "highlight": "一句话核心动态",
      "data_numbers": [
        {{"content": "数据内容", "context": "背景说明", "source": "来源文章"}}
      ],
      "events_news": [
        {{"content": "事件描述", "time": "时间", "parties": "相关方", "source": "来源文章"}}
      ],
      "insider_insights": [
        {{"content": "洞察内容", "type": "类型", "source": "来源文章"}}
      ],
      "sources": [
        {{"title": "文章标题", "key_contribution": "核心信息价值"}}
      ]
    }}
  ]
}}
```

## 示例

```json
{{
  "topics": [

        {{"content": "开源后GitHub星标突破10万", "context": "社区反响", "source": "极客公园"}}

      "events_news": [
        {{"content": "DeepSeek发布R1模型并开源", "time": "1月20日", "parties": "DeepSeek", "source": "量子位"}},

      ],
      "insider_insights": [
        {{"content": "据内部人士透露，DeepSeek团队仅200人，但效率极高", "type": "内幕消息", "source": "猫笔刀"}},

      ],

        {{"title": "量子位《DeepSeek R1深度解析》", "key_contribution": "技术架构和成本分析"}},

```

## 注意事项

1. **必须输出合法 JSON** - 不要在 JSON 外添加任何文字
2. **每条信息标注来源** - source 字段必填
3. **宁多勿漏** - 多提取一些信息比漏掉关键信息好
4. **类型字段** - insider_insights 的 type 可选：内幕消息/独家判断/趋势预测/争议观点
